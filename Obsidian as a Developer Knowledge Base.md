I've tried Notion, Roam, Logseq, and plain markdown folders. Obsidian is where I stayed — but not for the reasons most people cite. This video is about using Obsidian specifically for engineering work: code snippets, architecture decisions, debugging notes, and system design.

## My actual setup

- Daily note as a running work log (replaces Jira comments)
- Project folders synced via git to stay device-agnostic
- Dataview for surfacing stale TODOs and open questions
- Template for Architecture Decision Records
- Why I don't use Canvas or plugins that need the internet

The vault is public on GitHub — I'll link it in the description.
